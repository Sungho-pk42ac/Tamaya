02 Backend — Clean Architecture 4 레이어

의존성은 항상 안쪽(domain)을 향합니다: presentation → application → domain ← infrastructure. infrastructure는 domain이 정의한 인터페이스를 구현할 뿐, domain은 infrastructure를 모릅니다(의존성 역전).

04 요청 흐름 — 대화에서 일기까지

사용자가 AI와 대화하고, 대화를 일기로 확정(finalize)하면 청크화·임베딩되어 저장되는 핵심 경로입니다.

sequenceDiagram autonumber participant U as 사용자 participant R as chat_router participant UC as send_message participant AG as chat_agent participant SVC as ai_chat_service participant CL as CLOVA (mock) participant FIN as finalize_diary participant EMB as embedding_service participant DB as PostgreSQL + pgvector U->>R: POST 메시지 R->>UC: execute(session, content) UC->>AG: 대화 처리 AG->>SVC: AI 응답 요청 SVC->>CL: chat completion CL-->>SVC: 응답 텍스트 UC->>DB: chat_message 저장 Note over U,DB: 하루를 마무리하며 일기 확정 U->>FIN: 일기 확정 요청 FIN->>EMB: extract_chunks → 임베딩 EMB->>DB: diary + event_chunk(vector) 저장

CLOVA_MOCK_MODE=true가 기본이라 API 키 없이도 위 흐름이 끝까지 동작합니다.
